# Pokédex Tracker — Project Reference

## What It Is
A single-page React app for tracking Pokémon collection progress across Gen III games (Ruby, Sapphire, Emerald, FireRed, LeafGreen). Includes two utility tools: a Gen III/IV catch rate calculator and a Gen III IV checker with session persistence.

## Architecture

### Component Tree
```
App
├── Layout (tracker tab)
│   ├── Header  ← tab bar + all controls
│   ├── BoxView | ListView
│   └── DetailPanel (sidebar desktop / overlay mobile)
├── CatchCalculator (catch-calc tab)
├── IvChecker (iv-checker tab)
└── RouteInfo (routes tab)
    └── RouteDetail (encounter table, shown when a route is selected)
```

### Three Zustand Stores (all persisted to localStorage)

| Store | Key | What it holds |
|-------|-----|---------------|
| `useSettingsStore` | `pokedex-settings-v1` | viewMode, dexMode, activeGames, activeGeneration, darkMode, availabilityMode, showUncaughtOnly, activeTab |
| `useDexStore` | `pokedex-tracker-v1` | caught[], pending[] (arrays of Pokémon IDs) |
| `useIvStore` | `iv-checker-pc-box-v1` | savedSessions[] (IV checker entries) |

`searchQuery` and `selectedPokemonId` are **intentionally not persisted** — losing them on refresh is the desired UX.

### Data Loading
`App.tsx` fetches a **single file** at startup:

```
/data/pokemon.json  (1.2 MB compact JSON, served from public/)
```

All 386 Pokémon are in one object with everything the app needs: encounters, evolution chains, regional dex entries, catch rate, and base stats. `meta.json` and `boxes.json` are **statically imported** (small, never change at runtime).

### Filtering Pipeline (`src/hooks/usePokemonFilter.ts`)
```
allPokemon (386)
  → slice by generation range (meta.generations[activeGeneration].pokemonRange)
  → if regional dex: sort by regional number
  → determine relevant games (activeGames || all gen games)
  → build availability sets:
      directlyCatchable   — has encounters in relevant games
      npcTrade            — in npc-trades.json for relevant games
      obtainable          — catchable + trades + full evolution chain + breeding babies
  → apply availabilityMode (all / obtainable / catchable)
  → apply searchQuery filter
  → attach display metadata (displayNumber, isHighlighted, isVersionExclusive, exclusiveGames)
→ filteredPokemon
```

### Dark Mode
- **Pre-render init**: `src/main.tsx` reads `pokedex-settings-v1` from localStorage and applies/removes `dark` class on `<html>` before React mounts (prevents flash).
- **Runtime toggle**: `useSettingsStore.toggleDarkMode()` calls `applyDarkMode()` which manipulates `document.documentElement.classList` directly.
- **No React effect needed**: `App.tsx` does NOT sync dark mode — the store action handles it.

| File | Location | Generated by | Contents |
|------|----------|-------------|----------|
| `pokemon.json` | `public/data/` | `scripts/fetch-data.ts` | 386 Pokémon: id, name, types, sprites, flags, catch rate, base stats, encounters per game, evolution chains, regional dex entries. **Compact JSON (no indent).** |
| `boxes.json` | `src/data/` | `scripts/fetch-data.ts` | 13 PC boxes × 30 slots, with labels and ID lists |
| `meta.json` | `src/data/` | `scripts/fetch-data.ts` | Generation definitions, regional dex metadata (Hoenn 202, Kanto 151), generation timestamp |

Regenerate with `npm run generate-data`. Script batches 10 Pokémon per request with 250ms delay to avoid PokéAPI rate limits.

## Key Conventions

### Type Colors
Shared constants live in `src/lib/type-colors.ts`:
- `TYPE_COLORS` — hex strings for inline CSS (`style={{ backgroundColor }}`)
- `TYPE_BG_COLORS` — Tailwind `bg-*` classes for `className` usage

Do **not** redefine these locally in components.

### activeGames = [] means "All Games"
An empty `activeGames` array in the settings store means no filter is applied (show all games). This is by design. The `noneSelected` variable in `GameSelector.tsx` is `true` when `activeGames.length === 0` — it highlights the "All" button.

### Version Exclusivity
Exclusivity is **context-sensitive** — computed dynamically in `usePokemonFilter.ts` based on what games the user has selected, NOT a static field from the JSON.

**Rule**: A Pokémon is `isVersionExclusive` when:
- At least 2 games are selected (`activeGames.length > 1`), AND
- The Pokémon has wild/gift/static encounters in **exactly 1** of those selected games

`FilteredPokemon.exclusiveGames: string[]` holds the game(s) the Pokémon IS in (among selected), used to display "★ FireRed only" in the DetailPanel badge.

### BestSpot (encounter highlight)
The "best spot" summary shown in the DetailPanel's `LocationTable` is **computed at render time** by `computeBestSpot()` inside `LocationTable.tsx`. It is not stored in `pokemon.json`.

### Encounter Method Mapping
PokéAPI encounter method slugs are normalized to display strings in `scripts/fetch-data.ts → mapEncounterMethod()`. Location area slugs are formatted by `src/lib/format-location.ts` which has hardcoded overrides for Hoenn and Kanto locations. Add new overrides there when expanding to new generations.

### PC Box Layout
Boxes are 6 columns × 5 rows = 30 slots. 13 boxes cover all 386 Gen 1–3 Pokémon (386 ÷ 30 = 12.87, rounded to 13). Box assignments are in `src/data/boxes.json` and are generated in order (box 1 = #001–#030, etc.).

## Scope: Currently Gen III Only

`meta.json` lists `activeGenerations: [3]`. The type system supports Gen 1–9 (`GameVersion` union in `src/types/index.ts`) but only Gen 3 data is fetched and displayed.

**Expansion path**: To add a new generation, update `scripts/constants.ts` to include the new version names, add generation metadata to `GEN_META`, run `generate-data`, and add display colors/labels to `GAME_COLORS`/`GAME_LABELS` in `src/types/index.ts`. Location overrides for the new region need to be added to `src/lib/format-location.ts`.

## Known Deferred Items

- **No ErrorBoundary** — app goes blank on unhandled render errors. Low priority.
- **Evolution algorithm is O(n²)** in `usePokemonFilter.ts` — runs over ~386 Pokémon max, not a real perf concern.
- **Location overrides incomplete** — `src/lib/format-location.ts` only has Hoenn/Kanto overrides. Expand when new gens are added.
- **No input validation on JSON imports** — data files are cast directly (e.g. `pokemonData as Pokemon[]`). Safe as long as `generate-data` produces correct output.
- **Ruby/Sapphire bad-poison bug** — The catch rate calculator uses the corrected formula for all games (as noted in `src/lib/catch-rate.ts`). The RS-specific behavior is not modeled.
